Kanban

Concepto:

La metodología kanban se fundamenta en una filosofía centrada en la optimización continua, donde las labores se "extraen" de una lista de actividades pendientes en un flujo de trabajo constante.

Los 4 principios Kanban

1.) Empezar con lo que hace ahora.
2.) Comprometerse a buscar e implementar cambios incrementables y evolutivos.
3.) Respetar los procesos, las responsabilidades y los cargos actuales.
4.) Animar el liderazgo en todos los niveles.

La 6 practicas de Kanban

1.) Visualizar el flujo de trabajo.
2.) Eliminar las interrupciones.
3.) Gestiomar el flujo.
4.) Hacer las políticas explícitas (Fomentar la visibilidad) etc.

Jenkins

¿Que es?

Es un servidor de integración continua automatizado de código abierto que puede organizar una secuencia de actividades para ayudar a automatizar la integración continua (y más). Al usar Jenkins, los desarrolladores pueden acelerar el proceso de desarrollo de código; porque Jenkins puede automatizar, simplificar y acelerar todos los procesos de construcción y prueba de su proyecto. Además, Jenkins se puede implementar durante todo el ciclo de vida del desarrollo.

Con Jenkins, las organizaciones pueden acelerar el proceso de desarrollo de software a través de la automatización. Integra procesos de ciclo de vida de desarrollo de todo tipo, incluidos compilación, documentación, prueba, empaquetado, etapa, implementación, análisis estático entre otros.

Funciones De Jenkins

- Control de versiones: Es la practica del seguimiento y la gestión de cambios en el código del programa. Cuando un desarrollador edita el código, el sistema de control de versiones toma una instantánea de los archivos. Luego guarde esta instantánea de forma permanente para que pueda restaurarla más tarde si es necesario.
- Entrega continua: un enfoque en el que los equipos producen software dentro de un ciclo de vida corto, lo que garantiza que el software se pueda entregar en cualquier momento y, si es necesario, manualmente.
- Implementación continua: se refiere a la publicación automática de los cambios implementados por los desarrolladores desde el repositorio hasta la producción, poniéndolos a disposición de los clientes. Adoptar la implementación continua significa que puede ofrecer la experiencia de usuario más rápida posible sin sacrificar la calidad.

Beneficios

- Plataforma independiente. Jenkins está disponible para todas las diferentes plataformas y sistemas operativos, ya sea OS X, Windows o Linux.
- Fácil soporte Dado que es de código abierto y ampliamente utilizado, no tiene escasez de soporte de grandes comunidades en línea formadas por equipos ágiles.
- Es de código abierto, fácil de usar, fácil de instalar y no requiere instalación ni componentes adicionales.

Ventajas De Usar Jenkins

- Jenkins está dirigido por una comunidad muy abierta. Realizan reuniones públicas mensuales y reciben información del público sobre el desarrollo del proyecto Jenkins.
- Jenkins también es compatible con la arquitectura en la nube para que pueda implementar Jenkins en plataformas en la nube.
- La razón por la que Jenkins se volvió tan popular es porque fue creado por un desarrollador para desarrolladores.

Desventajas De Usar Jenkins

- Una de las razones por las que muchas personas no implementan Jenkins es la dificultad de instalar y configurar Jenkins.
- Su interfaz está desactualizada y no es fácil de usar en comparación con las tendencias actuales de la interfaz de usuario.
- Aunque muchos desarrolladores como Jenkins no son fáciles de mantener, ya que Jenkins se ejecuta en un servidor y requiere algunas habilidades de administración del servidor para realizar un seguimiento de su actividad.

Motores De Bases De Datos

Es el principal servicio de almacenamiento, procesamiento y protección de datos. El motor de base de datos proporciona acceso controlado y procesamiento rápido de transacciones para satisfacer las necesidades de las aplicaciones empresariales más exigentes. Utilice un motor de base de datos para crear una base de datos relacional para el procesamiento de transacciones en línea o el análisis de datos en línea. Puede crear tablas para almacenar datos y objetos de base de datos como índices, vistas y procedimientos almacenados para ver, administrar y proteger sus datos. Teniendo en cuenta la gran cantidad de información almacenada en la base de datos, así como el hecho de que cada vez más aplicaciones y personas piden usarla, los motores de base de datos nacieron como una alternativa para optimizar el proceso de acceso, consulta y extracción o implementación. información de una base de datos. , creando así un entorno más sencillo, agradable y eficiente para su uso.

Los motores llevan unos parámetros a cumplir los cuales son:

Abstracción de información, independencia, redundancia mínima, consistencia, integridad de seguridad, respaldo y restauración, control de concurrencia, tiempos de respuesta rápidos.

Principales Motores

◈ MySQL: es el gestor de bases de datos más utilizado en el mundo. Es un sistema multiusuario y multiproceso utilizado en la mayoría de los sitios web modernos y aplicaciones gratuitas. Entre las principales ventajas del gestor de bases de datos MySQL se encuentran la facilidad de uso y el buen rendimiento. Además, es muy fácil de instalar y configurar, soportando multiplataforma.

◈ MariaDB: Básicamente el mismo software que el motor MySQL. Cuando Oracle adquirió MySQL (en realidad adquirió Sun Microsystems), Michael Widenius, uno de los fundadores de MySQL, comenzó a desarrollar el software él mismo (técnicamente una bifurcación) creando MariaDB. Totalmente compatible con MySQL, puede reemplazarlo fácilmente. Es una buena opción para aquellos que buscan una independencia completa de Oracle y una mayor "pureza" del código fuente abierto.

◈ Oracle: este es un software propietario, lo que significa que debe comprar una licencia para usarlo. Además del modelo relacional clásico (XML, JSON, RDF), proporciona transacciones ácidas y datos parcialmente estructurados. En general, es una herramienta muy confiable y efectiva, y también es una buena opción si tiene un presupuesto limitado, ya que los costos de licencia pueden ser altos.

◈ SQLite: Es una biblioteca C que permite transacciones de datos. Su principal ventaja es que no requiere el uso de un servidor o configuración por lo que ocupa mucho menos espacio que otros gestores. Además, ofrece un buen desempeño y cumple con los criterios de atomización, integridad, aislamiento y durabilidad.

◈ PostgreSQL: este motor es uno de los más utilizados en el mundo y, con mucho, el motor relacional de código abierto más avanzado. Generalmente se considera una alternativa a MySQL. También admite transacciones ácidas e incluye un optimizador de consultas avanzado y mecanismos de respaldo muy confiables. Nuevamente, esta herramienta es intrínsecamente relacional, por lo que puede no ser adecuada para casos de uso como redes sociales u otros que requieren modelado de gráficos.

Soap (Simple Object Access Protocol)

Es un protocolo ligero para el intercambio de información en entornos distribuidos y descentralizados. Los mensajes SOAP son la transmisión de información del remitente al receptor. Los mensajes SOAP se pueden combinar para crear patrones de solicitud/respuesta. También es independiente del transporte, pero a menudo se transfiere a través de HTTP para trabajar con la infraestructura de Internet existente. Se necesita jabón para los servicios de Internet, las interfaces a través de las cuales un dispositivo puede acceder al servicio de host. Los motores de búsqueda, las tiendas en línea y muchos otros servicios en Internet se ejecutan a través de dichos servicios de Internet, y SOAP es uno de los protocolos que pueden hacerlo.

El protocolo esta basado en XML y se compone de:

⇝ Sobre: El sobre define la estructura de la descripción del cuerpo del mensaje y cómo se maneja el cuerpo del mensaje. Es una construcción global para expresar cuál es el mensaje, a quién se envía y si es opcional u obligatorio.

⇝ Reglas de cifrado: defina reglas que permitan el intercambio de diferentes tipos de datos entre usuarios. Conjunto de reglas de codificación que representan versiones de un tipo de datos definido por la aplicación. Los estándares de codificación definen un mecanismo de serialización que se puede usar para intercambiar versiones de un tipo de datos definido por la aplicación.

⇝ La convención: para representar llamadas a procedimientos y respuestas.

Caracteristicas Principales:

⇝ Extensibilidad: seguridad y WS routing son extensiones
⇝ Neutralidad: bajo el protocolo de transporte TCP puede ser utilizado sobre cualquier protocolo de aplicación como HTTP, SMTP o JMS.
⇝ Independencia: permite cualquier modelo de programación.

Rest

Es una interfaz para vincular muchos sistemas basados ​​en HTTP (uno de los más antiguos) utilizados para recibir y generar datos y trabajar, devolviendo esos datos en formatos muy específicos como XML y JSON. REST se maneja a través de HTTP, los verbos que usa son exactamente los mismos, puede hacer GET, POST, PUT y DELETE con ellos. Aquí es donde entra la alternativa SOAP. REST resuelve esta complejidad agregada por SOAP, lo que facilita mucho el desarrollo de la API REST, en este caso un servicio donde almacenaremos nuestra lógica comercial y procesaremos datos con un montón de recursos de URL y cadenas de datos que limitaremos, luego nuestro BACKEND lo hará. será nuestra lógica de negocio pura la que utilizaremos.

API REST: esta es una interfaz de programación de aplicaciones (API o API web) que se adhiere a las restricciones de la arquitectura REST y permite la interacción con servicios web compatibles con REST. El informático Roy Fielding es el creador de las Transiciones de Estado Representativo (REST). Las API son las definiciones y protocolos utilizados para desarrollar e integrar el software de aplicación. En otras palabras, la interfaz API le permite interactuar con su computadora o sistema para obtener datos o realizar funciones para que el sistema comprenda la aplicación y corresponda a ellos.

Caracteristicas de Rest

⇉ Protocolo cliente/servidor sin estado: cada solicitud HTTP contiene toda la información necesaria para completarla, lo que significa que ni el cliente ni el servidor necesitan recordar ningún estado anterior para poder responderla. Si bien esto es cierto, algunas aplicaciones HTTP tienen almacenamiento en caché incorporado. Se ha establecido el llamado protocolo cliente-caché-servidor sin estado.

⇉ Interfaz unificada: para transferir datos en un sistema REST, ciertas acciones (POST, GET, PUT y DELETE) se aplican a los recursos si están definidos por un URI. Esto facilita tener una única interfaz que organiza el proceso de información.

⇉ Sistema de capas: Arquitectura jerárquica entre los componentes. Cada una de estas capas lleva a cabo una funcionalidad dentro del sistema REST.

Ventajas De Rest

⇉ Esto nos permite separar el cliente del servidor. Esto significa que nuestro servidor se puede desarrollar en Node y Express, y nuestra API REST, como Vue, no tiene por qué ser la misma.

⇉ Visibilidad, confiabilidad y escalabilidad. Separar el cliente y el servidor tiene una clara ventaja, y es que cada equipo de desarrollo puede escalar fácilmente el producto, migrar a otros servidores o realizar cualquier cambio en la base de datos, siempre que cada grupo solicite correctamente los datos.

⇉ Es totalmente independiente de la plataforma, así que podemos hacer uso de REST tanto en Windows, Linux, Mac o el sistema operativo que nosotros queramos.

Desventajas De Rest

⇉ La seguridad es una deficiencia y puede llegar a ser una tarea muy difícil de implementarla correctamente.

⇉ Puede tener varios servidores en un esquema REST, algunos de los cuales desconocen la existencia de otros. No sabes si el usuario está logueado o no en el servidor y si le estás enviando ciertos datos. Tampoco sabe en qué servidor podría terminar la solicitud.

En informática, un tipo de datos abstractos (ADT) o tipo de datos abstractos (ADT) es un modelo matemático que consiste en una colección de operaciones definidas en un conjunto de datos en el modelo.

Estructura de Datos Lineales

Son aquellos elementos que ocupan posiciones consecutivas en la estructura, teniendo cada elemento un sucesor y un antecesor.

Estructura de Datos Jerárquicas

Es un modelo de datos en el que los datos se organizan en una estructura similar a un árbol.
